\ hygen 小工具 /

今天是要來提一個小工具
我覺得在產元件的時候很方便

有時候會覺得每次都要在新增一個元件後
再新增一個 storybook 檔
也許還有 test 檔
座一個元件 就要多做兩三個元件
多做幾個 就要多做幾倍的元件 ＱＱ
真是麻煩～

這時候如果有工具可以讓我在做一個元件的同時
幫我把另外的元件也產出來的話
那該有多好～～～

然後我就無意間發現了 hygen 這個工具
他可以用下指令的方式 產出一個元件
然後！ 你可以客製化的把另外想要的元件也產出～～

花了一點時間 去寫他的設定檔
以下是分享

hygen 的使用

我都是用 npx hygen new components --name Component 的方式使用他的
首先在 root 資料夾裡 放一個.hygen.js 檔案 裡面是要告訴 hygen 指令 要去哪邊找模板

// .hygen.js
module.exports = {
  templates: `${__dirname}/.hygen`,
};

接著在 root 資料夾裡 放一個叫做 .hygen 的資料夾
底下再放一個 new/ 的資料夾 => 作為我產生元件時要下的指令時，他會去找這個資料夾底下的設定
/new 底下我會再放分類的資料夾 => 作為這次要產出什麼樣的檔案的分類， 例如: .md 檔 or .js 檔 以元件為例： 在/component 資料夾裡放一個 index.js.ejs.t 的檔案

// index.js
module.exports = {
  prompt: ({ inquirer }) => {
    const questions = [
      {
        type: "input",
        name: "path",
        message: "What is the path of the Component?",
      },
    ];
    return inquirer.prompt(questions).then((answers) => {
      const { path } = answers;
      const absPath =
        path.length === 0 ? "src/components" : `src/components/${path}`;
      return { ...answers, absPath, path };
    });
  },
};

// index.js.ejs.t
---
to: <%= absPath %>/<%= name %>/index.tsx
---

import * as React from "react";
import { Link } from "gatsby";
import styled from "styled-components";

const Layout = styled.div.withConfig({ componentId: "<%= name %>_Layout" })``;

type <%= name %>PropTypes = {
};

export default function <%= name %>(props:<%= name %>PropTypes) {
  return <Layout></Layout>;
}

其中 index.js 檔 會在下完指令後 在 cmd 裡面變成一條問句 What is the path of the Component?
然後 輸入的答案會變成 path 這個變數的值
最後我把它組成 absPath 傳出去
就會變成 <%= absPath %> 這樣子的變數可以使用
在 index.js.ejs.t 這個檔案裡面
可以先把你通常會用的元件內容 做一個模板出來
例如我上面的範例 是用來做一個 React 的模板

接著 在同樣/component 的資料夾裡 再多放一個 stories.js.ejs.t
這個就是會做一個 storybook 檔案 在旁邊
附上我的範例

---
to: <%= absPath %>/<%= name %>/<%= name %>.stories.tsx
---

import * as React from "react";
import <%= name %> from "./";
import { Meta, Story } from "@storybook/react";


export default {
  title: "Example/<%= !path ? 'components' : 'pages' %>/<%= name %>",
  component: <%= name %>,
} as Meta;

const Template: Story = (args) => <<%= name %> {...args} />;

export const Primary = Template.bind({});
Primary.args = {};

現在就可以使用指令來產生檔案了～～～～
就是用我上面提到的 npx 指令 npx hygen new component --name Component
就是叫 hygen 去 new 這個資料夾底下的 component 底下找 index 去執行產元件
cmd 會問你要不要加上 path ？
如果沒有填任何東西的話 就會在設定好的 absPath 也就是 src/components 底下直接放一個資料夾 叫做 Component(你取的 name)
裡面會有一個 index.tsx 檔案 跟一個 stories.tsx 檔案
學會自己寫設定檔之後 元件就可以想怎麼產了～～～
