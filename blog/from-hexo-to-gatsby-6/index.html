Gatsby用起來_發布到github
September 20, 2022

是時候把部落格發布出去了

發布嘛～
好像沒很難ＸＤ
現在也有很多平台提供發佈功能
例如 netlify、heroku 之類的
這次我選的是 github-pages
在官網裡也有詳細的說明
how-gatsby-works-with-github-pages
基本上就照著說明做
也能做個大概出來ＸＤ

需要安裝套件 gh-pages
npm install gh-pages --save-dev
然後把專案指向 github 上的 repo
git remote add origin htpps://github.xxxxxxxxxxx
最後 部署就是跑指令
npm run deploy 這個指令是要自己建立在 package.json 裡面的

// 在package.json 裡的scripts 多加一行
scripts: {
  //... some scripts
  "deploy": "gatsby build && gh-pages -d public -b main"
}

這行指令會執行 build 後用 gh-pages 套件把網站放到 main 這個分支 確保在 github 裡的這個 repo 中的設定
是把 github-pages 分支設定成 main
你要改成 master 也行，記得 script 裡也要改成一樣的就行

最後應該就沒問題了唷 ～～

Support for password authentication was removed on August 13, 2021

補充一個我遇到的問題 ＱＱ
就是 github 的密碼認證已經不能用之類的
現在改採用 token 認證的方式

所以要改用 token 呈現的 github-url 來上傳 gh-pages

首先進到 github 首頁 > 到個人頁面設定 Setting > 選擇 Developer settings > personal access token > 按右上角的 generate new token > 接著把產生出來的 token 記下來

到 cmd 裡 重下個指令 git remote set-url origin https://<TOKEN>@github.com/.....
接著應該就可以正常的部署了
