從 js 到 ts 的過程

但這次在轉換的過程中

覺得滿好的

但在轉換的過程中還是時不時噴一些錯
以下就分享一下 可能會犯的錯ＸＤ

  • Type '{}' is missing the following properties from type

如果在 prop 裡面沒有設定可以不給這個參數的時候
會出現這個錯誤訊息
指的是 你沒有放著這個參數在 propType 裡面
解決辦法當然就是記得放 prop
但如果是不一定要放的 prop
可以在宣告 type 的時候加上一個?

type TYPE = {
  prop_necessary: string,
  prop_unnecessary?: string,
};
  • cannot be used as a JSX component. Its return type 'ReactNode' is not a valid JSX element. Type 'undefined' is not assignable to type 'Element | null'.

這是在 render jsx(tsx)的時候
如果沒有正確的放 return 裡面的東西會噴的錯
例如我這樣寫的 :

// some Component

//  type from prop
const renderPage = {
  type1: <>render this page1</>,
  type2: <>render this page2</>,
};

return renderPage[type];

// 但應該要這樣寫

return <>{renderPage[type]}</>;

一樣可以使用 enum 的方式來產不同類型的頁面
但需要在最外面加上一層包起來

其實大多數的問題都可以用錯誤訊息來找到解法
所以也不用太擔心沒辦法寫好 🤷‍♂️
