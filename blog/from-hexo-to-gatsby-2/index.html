關於開發 Gatsby

這部分的話 就跟大多數的 react 專案差不多了～

我這邊開發 基本上都是照著官網建議 或者是看起來比較多人用的 plugin 在做

基本上專案架構會是這樣的
my-gatsby-site (root folder)
├─ src
│ ├─ images
│ │ └─ icon.png
│ └─ pages
│   ├─ 404.js
│   └─ index.js
├─ README.md
├─ gatsby-config.js
├─ package-lock.json
└─ package.json

src 資料夾通常會放元件/頁面... 等資料
gatsby-config 是調整plugins會用到的設置檔

然後還有一些檔案是需要自己看需求調整新增的：
gatsby-browser 可以做一個container layout
layouts/index.js 我新增一個layouts的資料夾來放一些樣式的檔案
  ex: reset.css & theme

gatsby /pages

在 /pages 資料中
檔案名稱就代表了網址路徑
例如 index.js 就是 首頁
about.js 就是對應到 網址/about 這一頁
所以可以直接藉由建立新的檔案 來建立新的網址頁面
非常方便

gatsby 會在產生頁面的時候
找到相對應的檔案 建立起來
如果沒有找到的話 就會引導到 404.js 這個頁面去

gatsby 動態路徑

首先先上官網
裡面有詳細的說明 讚讚

這次要做的是把 md 格式的文章轉化成網頁的頁面
所以會需要一個動態路徑
比如說 ： md 文章有個 id 叫 abc123
讓他對應一個 /path/abc123 的網址

要建立動態的路徑 首先要有一個能夠動態產網址的檔案
這部分 不論是 gatsby 或 next.js 的做法大同小異
在建立檔案的時候 用大括號包起來
=> {MarkdownRemark.frontmatter__path}.js
這樣就能吃到動態路徑了～ 這個 MarkDownRemark 是一個 plugin 的用法
為了吃到 md 格式 必須先裝這個套件 npm i --save gatsby-transformer-remark

可以參考這裡 詳細的說明

然後需要調整一下 gatsby-config 的設置

module.exports = {
  siteMetadata: ...,
  plugins: [
    ...,
    {
      resolve: `gatsby-source-filesystem`,
      options: {
        name: `md_blog_posts`,
        path: `${__dirname}/src/md_blog_posts`, // 這裡是你存放md的資料夾名稱
      },
    },
  ],
};

然後因為 gatsby 內建用 graphQL
所以在這頁下面用 graphQL 的方式 來取資料
不過別擔心
官網裡面已經有示範大概怎麼寫了
照著做沒問題的
你也可以進 localhost:8000/___graphql 裡面去練習打資料

下面是我的頁面作法：
我的動態路徑是直接放在/pages 底下的
所以要在裡面判斷是哪種類型的 md 文章

當然另外的方法是 在 pages/底下 先新增一個資料夾 例如： blog/ 底下放 index.js 與 {動態的 md_path}.js
這樣一來 走到/blog 這頁會先出現 index.js
然後 /blog/post1 就會是 post.md 這樣了

import React from "react";
import { graphql } from "gatsby";
import BlogPost from "@components/page/BlogPost";
import ProjectPost from "@components/page/ProjectPost";

export default function Template({
  data, // this prop will be injected by the GraphQL query below.
}) {
  const { markdownRemark } = data; // data.markdownRemark holds your post data
  const { frontmatter, html } = markdownRemark;
  const { type } = frontmatter;

  const renderPost = {
    blog: <BlogPost data={{ ...frontmatter, html }} />,
    project: <ProjectPost data={{ ...frontmatter, html }} />,
  };

  return renderPost[type];
}
export const pageQuery = graphql`
  query ($id: String!) {
    markdownRemark(id: { eq: $id }) {
      html
      frontmatter {
        date(formatString: "MMMM DD, YYYY")
        path
        title
        type
        image_cover
      }
    }
  }
`;
