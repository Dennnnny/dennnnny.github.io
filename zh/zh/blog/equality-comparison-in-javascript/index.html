Javascript怎麼做比較的
2022-10-19
圖片來源 unsplash

Javascript 裡的比較

最近重新看了一下 js 的一些基礎
然後對於這個 比較 才是更明白了一些

說到等於 『 = 』 這個符號
在 js 裡有兩種用法

第一種是賦值給變數
例如 let x = 1

第二種就是比較
比較又分兩種
比較嚴謹的 === 與不嚴謹的 ==
兩者的差異是

在嚴謹模式 === 下
比較值兩邊要 同時符合 相等的值與量同的型別 才會回傳 true
否則就是 false

而在 不那麼嚴謹的比較之下 ==
js 會偷偷幫你做型別的轉換
然後再拿來做比較
所以有時候會出現令人費解的比較結果 例如以下範例

[0] == ""; //false
[0] == 0; //true
[""] == ""; //true
[""] == 0; //true;

怎麼會 [0] == "" 是錯的 而其他的又都是對的呢 ！！
當知道原因後 就沒那麼難理解了

Js 偷偷在背後幫你做的型別轉換是有規則的
根據 MDN 的解釋
簡單來說（粗略的說明，詳細請參考 MDN）
Js 會想辦法把兩邊 變成同樣的型別來做比較

規則是

  • 在相同型別下
    參考這裡

  • 非相同型別比較

  1. 物件型別 vs 基本型別
    => 把物件轉成基本型別
    => 做法是 依序試試看兩種方式 : valueOf() / toString()
    => 如果 valueOf 沒有拿到基本型別的值 ， 則會使用 toString() 的方式轉換成 string 來比較

  2. 基本型別 - Boolean vs 基本型別 - 其他(String, Number)
    => 把 Boolean 轉成 Number (參考)

  3. 基本型別 - String vs 基本型別 - Number
    => 把 String 轉成 Number (參考)

轉化完後就可以進行比較了
以上面的例子來看

//1.
[0] == ""; //object vs string
// 因為 [0].valueOf() 還是[0] 所以會使用toString()
"[0]" == "";
// 然後比較出來等於 false

//2.
[0] == 0; // object cs number
// 同上
"[0]" == 0;
// 接著做字串轉成數字： "[0]" -> NaN,
NaN == 0; // 得到結果 false

// 其他的也可以依照這個邏輯來解答
